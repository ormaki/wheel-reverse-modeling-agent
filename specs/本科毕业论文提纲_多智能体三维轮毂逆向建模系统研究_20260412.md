# 本科毕业论文提纲

## 题目建议

基于多智能体协同的三维轮毂逆向建模系统研究

## 一、摘要

- 说明研究对象为轮毂 STL 到参数化 CAD/STEP 模型的逆向建模任务。
- 说明研究问题不是单次几何拟合，而是面向轮辋、轮心、PCD 孔、凹槽、辐条等结构的分阶段重建。
- 说明方法路线采用多智能体协同框架，将用户请求解析、任务编排、感知提取、参数建模、结果评估和迭代优化组织成闭环流程。
- 说明系统实现基于当前项目路径中的 `UserConsole`、`AgentRuntime`、`CoordinatorAgent`、`PerceptionAgent`、`ModelingAgent`、`EvaluationAgent`、`OptimizationAgent` 及可选 `LLMPolicyEngine`。
- 说明实验通过运行日志、点云/几何对比、可视化结果和阶段产物验证系统有效性。

## 二、目录提纲

### 第1章 绪论

#### 1.1 研究背景与意义

- 介绍轮毂逆向建模在工业设计、零部件复现和参数化重建中的应用价值。
- 说明 STL 网格模型难以直接编辑、难以约束复用，参数化 CAD 重建具有实际意义。
- 引出传统单流程建模方式在复杂轮毂结构下存在人工依赖强、试错成本高、稳定性不足等问题。
- 引出多智能体协同机制在任务拆解、流程组织和迭代修正方面的优势。

#### 1.2 国内外研究现状

- 概述 STL/点云逆向建模、参数化重建、多智能体系统在工程设计中的应用现状。
- 简述当前研究常见问题，包括几何特征提取不稳定、复杂结构建模困难、结果评估机制不足等。
- 说明现有工作多聚焦单算法或单模块，而较少面向完整闭环建模系统展开。

#### 1.3 研究内容

- 设计一个面向轮毂逆向建模的多智能体协同系统框架。
- 实现轮辋轮心回转体、PCD 孔、轮心凹槽和辐条等结构的阶段化建模流程。
- 引入运行时协调与评估优化机制，实现“感知-建模-评估-修正”的闭环。
- 在规则约束下引入 LLM 规划层，提升任务组织与参数调整能力。

#### 1.4 研究方法与技术路线

- 以当前项目中的多入口执行路径为基础，说明 `full`、`runtime`、`console`、`stage` 四类运行方式。
- 说明论文重点采用 `console -> runtime -> coordinator` 的真实智能体系统路径。
- 给出总体技术路线：用户请求输入、任务协议定义、协调分发、感知提取、建模生成、评估反馈、优化迭代、结果输出。

### 第3章 多智能体协同建模系统总体设计

#### 3.1 系统目标与设计原则

- 目标一：支持轮毂 STL 到参数化模型的自动化重建。
- 目标二：支持阶段化建模与运行时状态追踪。
- 目标三：支持在规则约束下的智能协调与优化迭代。
- 设计原则包括模块解耦、任务可追踪、结果可评估、失败可回退。

#### 3.2 系统总体架构

- 用户交互层：`UserConsole`
- 运行时控制层：`AgentRuntime`、`MessageBus`
- 协调控制层：`CoordinatorAgent`
- 感知建模执行层：`PerceptionAgent`、`ModelingAgent`
- 评估优化层：`EvaluationAgent`、`OptimizationAgent`
- 规划增强层：`LLMPolicyEngine`、`rulebook`
- 资产与状态层：`user_goal.json`、`state_snapshot.json`、`events.jsonl`、`results.json`、`llm_decisions.jsonl`

#### 3.3 智能体职责划分

- `UserConsole`：接收用户请求并生成 `UserGoal`
- `CoordinatorAgent`：统一调度各类内部智能体，控制流程起止
- `PerceptionAgent`：提取轮毂几何特征并输出结构化特征文件
- `ModelingAgent`：根据特征文件执行参数化建模
- `EvaluationAgent`：对建模结果进行评估和对比
- `OptimizationAgent`：根据评估结果生成下一轮调整建议
- `LLMPolicyEngine`：在规则约束下提供规划建议，不直接替代执行逻辑

#### 3.4 任务协议与消息传递机制

- 介绍 `UserGoal`、`AgentTask`、`AgentResult`、`RuntimeEvent`、`RuntimeStateSnapshot` 的作用。
- 说明系统如何通过消息队列驱动任务分发与结果回收。
- 说明协调智能体是唯一允许结束运行的节点。

#### 3.5 运行状态记录与可观测性设计

- 说明运行态快照、事件日志、结果日志和 LLM 决策日志的记录方式。
- 说明这些状态文件既服务调试，也可作为论文实验记录依据。

### 第4章 面向轮毂重建的多智能体协同方法

#### 4.1 整体闭环流程设计

- 用户请求进入 `UserConsole`
- 运行时创建任务并交由 `CoordinatorAgent`
- 协调智能体依次调度感知、建模、评估、优化模块
- 评估不达标时进入下一轮迭代，达标后输出最终结果

#### 4.2 感知阶段方法设计

- 说明感知目标是从 STL 中提取后续参数化建模所需的结构特征。
- 重点介绍轮辋轮心回转体、PCD 孔、轮心凹槽、辐条等结构的识别需求。
- 说明感知结果以结构化 JSON 形式输出，为下游建模提供依据。

#### 4.3 建模阶段方法设计

- 说明建模模块依据感知结果进行参数化重建。
- 说明系统采用分阶段建模思想，先稳定回转体，再加入孔、凹槽、辐条等复杂结构。
- 结合现有项目说明 `stage 01-04` 的阶段式实现思路。

#### 4.4 评估阶段方法设计

- 说明评估模块对模型结果进行质量判定。
- 介绍总体分数、可接受性判断、差异定位、可视化对比等输出内容。
- 强调评估不是附属步骤，而是驱动优化迭代的关键模块。

#### 4.5 优化阶段方法设计

- 说明优化模块根据评估结果给出下一轮感知参数和建模参数调整建议。
- 说明优化输出受白名单约束，避免系统失控。
- 说明当前项目中优化支持规则驱动与可选 LLM 规划增强两种模式。

#### 4.6 LLM 受约束协同机制设计

- 说明引入 LLM 的目的在于增强规划，而非替换既有建模程序。
- 说明 LLM 只能在既定执行顺序与允许参数范围内输出建议。
- 说明通过规则手册与白名单过滤，保证系统稳定性和可控性。
