# 本科毕业论文提纲

## 题目建议

基于多智能体协同的三维轮毂逆向建模系统设计与实现

## 一、摘要

- 研究对象为轮毂 STL 模型到参数化三维模型的逆向重建任务。
- 研究重点不是一次性生成完整模型，而是将轮辋轮心回转体、PCD 孔、轮心非孔特征和辐条拆分为可验证的阶段，逐步完成建模。
- 方法上构建了一个由用户交互、运行时调度、协调控制、感知提取、参数建模、结果评估和迭代优化组成的多智能体协同系统。
- 为提高系统可复现性，本文进一步设计了阶段任务描述与过程记录机制，用于约束每一阶段的目标、范围、交付物和验收标准。
- 实验通过阶段产物、日志记录和模型对比结果，验证该方法在轮毂逆向建模任务中的可行性。

## 二、目录提纲

### 第1章 绪论

#### 1.1 研究背景与意义

- 介绍轮毂逆向建模在零部件复现、结构分析和参数化设计中的应用价值。
- 说明 STL 网格模型虽便于获取，但难以直接编辑与复用，参数化模型更适合作为后续建模与优化基础。
- 说明轮毂结构中轮辋、轮心和辐条之间耦合较强，传统单流程建模方式容易出现试错成本高、过程不可追踪的问题。

#### 1.2 研究现状

- 简述逆向建模、参数化建模以及多智能体协同系统的基本研究现状。
- 说明现有工作往往偏重单一算法或单一模块，对“从输入模型到阶段产物再到评估修正”的完整闭环关注不足。

#### 1.3 研究问题

- 如何把轮毂逆向建模任务从“直接拟合一个结果模型”转化为“分阶段恢复关键结构”的过程。
- 如何构建一个既能拆分任务又能保持流程统一的多智能体系统。
- 如何通过阶段任务描述和阶段产物管理，提升建模过程的复现性和可追踪性。

#### 1.4 研究内容

- 设计面向轮毂逆向建模的多智能体协同系统框架。
- 设计轮辋轮心回转体、PCD 孔、轮心非孔特征和辐条的阶段化建模流程。
- 构建阶段任务描述与过程记录体系，使不同阶段能够按统一方式推进。
- 建立“感知-建模-评估-修正”的闭环运行机制。

#### 1.5 论文结构安排

- 第1章介绍研究背景、问题和研究内容。
- 第2章介绍系统总体方案与智能体构建方式。
- 第3章介绍阶段化建模方法与阶段推进方法。
- 第4章介绍系统实现、实验过程与结果分析。
- 第5章给出总结与后续展望。

### 第2章 系统总体方案与智能体构建

#### 2.1 系统目标

- 实现轮毂 STL 到参数化模型的自动化重建。
- 支持将复杂建模问题拆解为多个可单独验证的阶段。
- 支持运行状态记录、阶段产物管理和迭代修正。

#### 2.2 系统总体架构

- 用户交互层：负责接收用户请求，并将请求整理为系统目标。
- 运行时调度层：负责维护任务队列、共享状态和日志记录。
- 协调控制层：负责接收目标、分发任务、汇总结果和控制阶段推进。
- 感知建模执行层：负责几何特征提取和参数化模型生成。
- 评估优化层：负责比较建模结果、定位差异并生成下一轮调整建议。
- 结果记录层：负责保存阶段模型、运行日志、状态快照和实验记录。

#### 2.3 智能体划分依据

- 本系统的智能体划分不是按“概念”划分，而是按实际工作链路划分。
- 将用户输入、流程控制、特征提取、参数建模、结果评估和优化调整拆成不同智能体，是为了避免一个模块同时承担全部逻辑，便于调试和阶段复盘。
- 每个智能体只负责单一类型任务，而跨阶段推进和终止决策统一交由协调智能体处理。

#### 2.4 智能体构建方式

- 首先定义统一任务协议，包括用户目标、任务对象、结果对象和运行状态对象，使不同智能体之间可以通过一致的数据结构交互。
- 然后围绕实际工作流程构建六类核心角色：用户控制入口、协调智能体、感知智能体、建模智能体、评估智能体和优化智能体。
- 在执行过程中，不允许用户直接操作内部智能体，而是通过统一入口提交请求，由协调智能体负责分发和回收。
- 运行时调度模块负责维护任务队列与共享状态，使每轮执行都能够留下可复查的事件记录和结果文件。

#### 2.5 系统工作流程

- 用户首先通过控制台提交建模目标。
- 系统将目标转化为可执行任务，并进入运行时调度模块。
- 协调智能体根据当前阶段依次调用感知、建模、评估和优化智能体。
- 若评估结果不满足要求，则由优化智能体生成下一轮调整建议并重新进入流程。
- 若结果达到阶段要求，则保存当前阶段产物，并决定是否进入下一阶段。

### 第3章 阶段化建模方法与阶段推进方法

#### 3.1 阶段化建模思路

- 本文不直接追求一次生成完整轮毂模型，而是将任务拆分为四个阶段。
- 第一阶段完成轮辋轮心回转体，目的是先获得稳定基体。
- 第二阶段在稳定基体上补充 PCD 孔特征。
- 第三阶段补充轮心非孔特征，避免与孔位逻辑混杂。
- 第四阶段完成辐条生成，并在已有稳定主体上处理连接关系。

#### 3.2 阶段任务描述方法

- 为了保证四个建模阶段能够稳定推进，本文为每一阶段设计统一的任务描述结构。
- 阶段任务描述不只说明阶段名称，还同时限定该阶段的目标、输入、范围、约束、交付物和验收标准。
- 这种写法的作用在于把“当前阶段应该完成什么”和“当前阶段不能混入什么”同时明确下来。
- 通过统一的阶段描述结构，可以使回转体、PCD 孔、轮心非孔特征和辐条四个阶段保持清晰边界。

#### 3.3 阶段推进与验收方法

- 每个阶段完成后都需要输出对应模型文件和阶段记录文件。
- 只有当前阶段达到预期效果并被判定为稳定后，系统才进入下一阶段。
- 如果当前阶段未达到要求，则优先在本阶段内继续修正，而不是直接跳转到后续阶段。
- 对于失败阶段，也需要保留失败记录和剩余问题说明，为后续分析提供依据。

#### 3.4 感知与建模协同方法

- 感知阶段负责从 STL 模型中提取后续建模所需的关键结构特征。
- 建模阶段根据感知结果进行参数化重建，且严格围绕当前阶段目标展开。
- 这种分工使得感知和建模既相互依赖，又能在日志和阶段产物中分开追踪。

#### 3.5 评估与迭代修正方法

- 评估阶段负责对比生成结果与目标模型之间的差异，形成质量判断。
- 优化阶段根据评估结果调整下一轮感知参数或建模参数。
- 若问题属于当前阶段内部问题，则优先在当前阶段收敛；若当前阶段未稳定，则不进入后续阶段。

### 第4章 系统实现与实验分析

#### 4.1 系统实现

- 介绍系统由用户控制台、运行时调度模块、协调智能体和各执行智能体共同组成。
- 介绍系统如何保存用户目标、运行快照、事件日志和阶段结果文件。
- 说明这些实现不仅服务运行，也服务论文中的过程展示与实验复现。

#### 4.2 阶段化建模实验

- 实验一：轮辋轮心回转体创建。
- 实验二：PCD 孔创建。
- 实验三：轮心非孔特征创建。
- 实验四：辐条生成。
- 每个实验均结合阶段提示词、阶段模型和阶段记录文件进行分析。

#### 4.3 结果分析

- 分析阶段拆分是否降低了复杂建模任务的耦合度。
- 分析多智能体协同机制是否提高了流程组织与问题定位效率。
- 分析提示词资产化方式是否提升了线程复现性和结果可追踪性。
- 分析当前系统在复杂辐条结构建模中的不足。

### 第5章 总结与展望

#### 5.1 工作总结

- 本文完成了一套面向轮毂逆向建模的多智能体协同系统设计。
- 系统将轮毂建模过程组织为阶段化、可追踪、可复盘的闭环流程。
- 提示词资产与阶段产物管理机制增强了系统的复现性和研究过程记录能力。

#### 5.2 后续展望

- 继续提高轮心复杂特征和辐条结构的感知与建模精度。
- 完善阶段评估指标与自动化对比方式。
- 在后续工作中继续提高系统的自动化程度和可扩展性。
