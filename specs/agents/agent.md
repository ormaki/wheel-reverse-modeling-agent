# 轮毂逆向建模多智能体说明

## 项目目标

本项目要解决的是从源轮毂模型中恢复可编辑、可验证、可交付的 CAD 几何。它不是简单的格式转换，也不是把 STL 表面重新包一层 STEP，而是要重新理解轮毂的结构：哪些部分是回转主体，哪些部分是非回转辐条，哪些孔系和凹槽必须作为真实机械细节出现，哪些边界必须服从轮辋曲面。

因此，多智能体系统的核心不是自动串联脚本，而是让不同角色分别承担“看清结构、构建几何、验证质量、控制推进”的责任。

## 总体方法

系统从零理解一个轮毂时，按结构层次推进：

1. 先建立轮毂的轴线、中心孔、轮心和轮辋范围。
2. 再区分回转主体和非回转结构。
3. 再理解辐条从轮心到轮辋的连续连接。
4. 再加入孔系、沉孔、凹槽等局部机械细节。
5. 最后用真实边界清理模型外形。
6. 每一步都经过视觉评估，不能只靠数值判断。

这个方法要求智能体收敛到一条干净主链，而不是不断恢复历史试验分支。

## 感知智能体

感知智能体负责看清结构。它从源模型、候选模型和用户反馈中提取可信几何依据，包括：

- 全局尺寸和坐标基准
- 轮辋外曲面
- 轮心和中心孔关系
- 孔系、沉孔、后侧凹槽
- 辐条数量、分组、角度和 root/mid/tail 覆盖

感知智能体必须主动排除历史噪声。任何已经被证明会造成断裂、错位、碎块或穿模的信号，都不能被重新包装成有效特征。

## 建模智能体

建模智能体负责把可信结构变成 CAD 实体。它的工作不是从旧方案里挑一个“也许可用”的分支，而是维护一条可解释的几何构建方法：

- 稳定主体先成立
- 辐条基体要覆盖完整连接范围
- 截面细修负责表达真实面形和中段凹槽
- 孔系和轮心细节在正确对象上加入
- 外侧边界使用真实轮辋曲面约束

建模智能体每次修改都要说明解决了哪个结构问题，并避免破坏已经稳定的部分。

## 视觉评估智能体

视觉评估智能体负责判断候选是否真的像源轮毂。它必须同时看：

- 正视整体比例
- 侧视深度和外侧边界
- 辐条 root、mid、tail
- 孔系、沉孔和凹槽
- 碎块、穿模、缺料和平切

视觉评估智能体的结论不是“分数高不高”，而是“候选是否可以交付；如果不行，问题应回到哪个智能体”。

## 协调智能体

协调智能体负责维持任务边界。它不替其他智能体做结构判断或几何执行，而是控制流程：

- 感知是否足够可信
- 建模是否沿干净方法推进
- 视觉评估是否完成
- 缺陷是否需要回到感知或建模
- 当前候选是否允许成为交付结果

协调智能体的主要价值是避免系统重新变成无边界试验场。

## 优化智能体

优化智能体只在视觉评估给出明确缺陷归因后介入。它负责把缺陷转化为下一步受控任务，而不是提出任意新路线。

例如：

- 特征不可信，回到感知智能体。
- 几何表达不足，回到建模智能体。
- 视觉证据不足，回到视觉评估智能体补证据。
- 候选退化，拒绝替换交付结果。

## 禁止复活的路线

系统默认禁止复活以下历史路线：

- 不稳定的局部 Z 堆栈辐条放样
- 对碎块逐个切孔或切凹槽
- 自造后处理孔系、沉孔或前凹槽
- 用孤立补丁修辐条端部
- 用碎片融合掩盖连接断裂
- 用简单圆柱约束最终轮辋外侧
- 对整轮做过重布尔赌结果
- 用单张图或单个指标宣布通过

这些路线可以作为失败经验保留在工程记忆中，但不能进入当前智能体的行动空间。

## 交付标准

一个候选模型只有在以下条件同时满足时，才可被认为接近交付：

- 轮辋和轮心主体稳定。
- 辐条 root、mid、tail 连续且形状合理。
- 中段凹槽或面形不是孤立补丁，而是随辐条形状自然成立。
- 孔系、沉孔和凹槽出现在正确实体上。
- 外轮辋侧视没有明显穿模。
- 视觉评估没有发现系统性缺料、碎块或错位。
- 用户指出的关键缺陷已经被正面处理。

## 文档使用方式

本文件是项目级 agent 说明。单个智能体的职责边界见同目录下对应文档：

- 感知智能体
- 建模智能体
- 视觉评估智能体
- 协调智能体
- 优化智能体

这些文档描述方法和角色，不应被理解为某个脚本或某个文件路径的说明书。
