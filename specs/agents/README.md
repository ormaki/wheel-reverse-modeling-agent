# 智能体说明索引

## 目录作用

该目录用于集中保存轮毂逆向建模项目中的智能体角色说明。

这些文档不是脚本清单，也不是历史方案索引，而是给后续智能体建立项目心智模型：这个项目要从源轮毂中恢复可编辑、可验证的 CAD 几何；每个智能体只承担自己边界内的判断和动作；所有推进都必须服务一条干净、可解释、可视觉验收的主链。

## 当前文档列表

- `agent.md`  
  从项目目标出发，统一说明多智能体系统为什么存在、如何分工、怎样协作。

- `perception_agent.md`  
  说明感知智能体如何看清轮毂结构、提取可信几何依据，并过滤已经证伪的历史信号。

- `modeling_agent.md`  
  说明建模智能体如何把可信结构转化为 CAD 实体，并维护稳定、可解释的几何构建方法。

- `visual_evaluation_agent.md`  
  说明视觉评估智能体如何从多视角判断候选模型是否真实接近源轮毂。

- `coordinator_agent.md`  
  说明协调智能体如何控制任务边界、组织角色协作和判断是否允许继续推进。

- `evaluation_optimization_agent.md`  
  说明优化智能体如何在明确缺陷归因后，把问题转成下一步受控修正任务。

## 组织原则

本目录采用以下原则：

- 先描述项目方法，再描述单个角色。
- 先说明结构判断，再说明执行动作。
- 先维护干净主链，再考虑局部修正。
- 不把历史失败方案、临时候选或单个工程文件写成当前方法。
- 不允许智能体通过恢复旧分支来扩大搜索空间。

## 使用方式

新的智能体接手项目时，应先阅读项目级说明，再阅读自己的角色说明。阅读目标不是记住某个路径，而是理解：

- 轮毂由哪些结构组成。
- 哪些结构必须连续。
- 哪些细节必须真实存在。
- 哪些缺陷说明方法出了问题。
- 哪些历史路线已经不能复活。
- 当前候选为什么必须经过视觉验收。

## 与其他资产的关系

智能体说明文档位于方法层。它们只描述职责、边界、协作关系和判断标准。

具体实现、协议对象、运行入口和产物位置属于工程层。工程层可以变化，但本目录中的角色说明应保持稳定，避免把智能体训练成“查找某个固定文件并照做”的执行器。

## 维护要求

更新本目录时，应优先回答三个问题：

1. 这个智能体负责什么结构判断？
2. 它能修改什么，不能修改什么？
3. 它如何知道自己的结果可以交给下游？

如果某段文字只是在指向某个历史文件、某个旧候选或某个临时脚本，而没有说明方法本身，就应该删除或改写。
