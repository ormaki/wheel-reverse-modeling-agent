# 智能体说明索引

## 1. 目录作用

该目录用于集中保存当前项目中各类智能体的角色说明文档。  
这些文档不是旧运行时主链的抽象说明，而是围绕“新线程按阶段接手主流水线”的执行方式整理出来的角色资产，用来说明各智能体在阶段推进中的职责、上下游关系、输入输出和边界。

## 2. 当前文档列表

- `coordinator_agent.md`  
  说明协调智能体如何组织阶段线程、约束阶段边界、控制阶段推进。

- `perception_agent.md`  
  说明感知智能体如何围绕当前阶段目标识别结构，并为建模阶段提供依据。

- `modeling_agent.md`  
  说明建模智能体如何只落实当前阶段允许处理的几何内容，并输出阶段产物。

- `evaluation_optimization_agent.md`  
  说明评估与优化智能体如何围绕当前阶段产物做验收判断，并决定是否继续停留在本阶段修正，或进入下一阶段。

## 3. 与其他资产的关系

当前项目中的智能体资产可以分成三层。

第一层是 skill 文档层，主要对应 `SKILL.md` 或 skill 脚本文件，用来说明某类能力负责什么、依赖什么、输出什么。  
第二层是 agent 说明文档层，也就是本目录中的各份 `agent.md` 文件，用来说明角色在阶段线程中的职责边界和协同关系。  
第三层是 protocol 协议层，主要对应 `models/agent_protocol.py` 等统一对象定义文件，用来说明不同角色之间用什么数据结构完成衔接。

## 4. 使用方式

如果需要画系统框架图、整理论文第 2 章角色说明、或者继续补充新的线程型智能体资产，可以优先按下面顺序阅读：

1. `prompts/NEW_THREAD_PROMPT.md`
2. `prompts/STAGE_INDEX.md`
3. 本目录中的各份 `agent.md`
4. `specs/agent_system_status.md`
5. `models/agent_protocol.py`

这样可以先看清阶段链，再看清角色边界，最后再看协议对象。

## 5. 当前组织原则

本目录采用的组织原则是：

- 角色划分要和当前线程接手主流水线的阶段链保持一致
- 每份文档都要说明职责、输入、输出、上下游和边界
- 不把旧主链的自动任务流直接当成当前主叙述
- 角色说明要能直接服务论文写作、框架图绘制和后续资产扩展
